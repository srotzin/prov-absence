**Cryptographic proof that an event did NOT happen.**

The dual of inclusion. If a Merkle tree proves "I saw X", `prov.absence` proves "X was never observed in this window."

---

## Why this exists

Modern AI systems need to assert negative facts:

- *"No call to model X was made in this audit period."*
- *"No tool call to a forbidden category occurred in this session."*
- *"No data was emitted to a sanctioned endpoint in this time window."*

Today these claims are made by fiat — an operator says "trust me." `prov.absence` makes them **cryptographically verifiable**: a lightweight, O(log n)-sized proof that any third party can check against a published Merkle root.

---

## How it works

A **sorted Merkle tree** commits to a set of observed events. Leaves are SHA-256 hashes of canonicalized event records, sorted lexicographically. Two sentinel leaves (`0x00...00` and `0xff...ff`) bracket all possible hash values.

To prove event X is absent, find the two adjacent leaves `L_i` and `L_{i+1}` in the sorted tree such that `L_i < hash(X) < L_{i+1}`, and produce inclusion proofs for both. Because:
1. Both leaves are cryptographically anchored to the root.
2. They are adjacent — no gap between them.
3. The tree is sorted — nothing with hash(X) can fit.

…the proof is complete. The verifier checks all three properties in O(log n) hash operations.

See [`SPEC.md`](./SPEC.md) for the full formal specification and security analysis.

## Quick start

```typescript
import { SortedMerkleTree, buildLeaf, verifyNonMembership } from "@hivecivilization/prov-absence";

// 1. Build a tree from observed events
const events = observedLLMCalls.map(e => buildLeaf(e.id, e.metadata));
const tree = new SortedMerkleTree(events);
const root = tree.root(); // anchor this root in your audit log

// 2. Prove a specific event was NOT observed
const proof = tree.proveNonMembership("llm-call:MALICIOUS:exfil", {
  model: "gpt-4o",
  destination: "sanctioned-provider.example.com",
});

// 3. Verify (stateless, can run anywhere)
const valid = verifyNonMembership(proof, "llm-call:MALICIOUS:exfil", {
  model: "gpt-4o",
  destination: "sanctioned-provider.example.com",
}, root);

console.log(valid); // true — cryptographically proven absent
```

## CLI

```bash
# Build a tree from a JSONL file (one { "eventId": "...", "data": {...} } per line)
prov-absence build events.jsonl
# → writes tree.bin, root.txt

# Prove an event is absent
prov-absence prove-absence "llm-call:MALICIOUS:exfil" > proof.json

# Verify the absence proof
prov-absence verify-absence proof.json "llm-call:MALICIOUS:exfil" $(cat root.txt)
# → exits 0 if valid, 1 if invalid
```

## Connection to SMSH-Stamp ecosystem

`prov.absence` is part of the **SMSH-Stamp** cryptographic provenance stack:

| Layer | Role |
|-------|------|
| `prov.stamp` | Time-stamped inclusion proof ("I saw X at T") |
| **`prov.absence`** | Non-membership proof ("X was never seen") |
| `prov.chain` | Linked window roots forming a verifiable audit chain |
| `prov.anchor` | On-chain or notarization anchoring of roots |

A non-membership proof is only as trustworthy as its root anchor. For production use, roots should be published to an immutable log (blockchain, transparency log, or notarized timestamp) before any dispute window opens.
